# 单页网站整合计划（更新版）

## 目标
将当前多页网站整合为单页设计，按照指定顺序排列内容，保持核心信息的同时简化展示。

## 页面结构（新顺序）

### 1. 主页介绍
- 保留当前的 hero 区域（标题和副标题）
- 简化背景设计，保持干净
- 保留简短的 workshop 介绍

### 2. 日程安排（Schedule）
- 整合 schedule.html 中的完整日程
- 保留时间线布局和详细安排
- 保持所有演讲者和活动信息

### 3. 演讲者列表
- 整合 speakers.html 中的所有演讲者信息
- 采用简洁的卡片式布局
- 每个演讲者包含：照片、姓名、职位
- **不包含**简短描述，只保留基本信息

### 4. Call for Papers
- 整合 call-for-papers.html 中的关键信息
- 保留主题领域和提交指南的核心内容
- 简化详细说明，突出重要日期
- 保留程序委员会信息

### 5. 组织者
- 保留当前的组织者信息
- 采用简洁的展示方式
- 放在页面的倒数第二部分

### 6. 赞助商
- 保留 XeonLabs 赞助商信息
- 保持简洁的展示方式
- 放在页面的最后

## 技术实现

### 1. 修改 index.html
- 移除导航栏中的多页链接
- 整合所有内容到单个页面
- 按照新的顺序排列各个部分
- 调整 CSS 确保响应式设计
- 保持现有的视觉风格和配色方案

### 2. 内容整合
- 从 schedule.html 提取完整的日程安排
- 从 speakers.html 提取演讲者基本信息（不含描述）
- 从 call-for-papers.html 提取征稿信息
- 保持信息的准确性和完整性
- 确保内容流畅过渡

### 3. 优化用户体验
- 保持页面加载速度
- 确保移动设备兼容性
- 提供清晰的视觉层次
- 突出核心信息和重要链接
- 确保各个部分之间的自然过渡

## 预期结果
- 单页网站，包含所有重要信息
- 内容排列顺序：主页介绍 → 日程安排 → 演讲者列表 → Call for Papers → 组织者 → 赞助商
- 简洁明了的设计，突出核心内容
- 演讲者信息简化，只显示基本信息
- 保持原有的视觉风格和品牌一致性
